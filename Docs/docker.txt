1. docker is a virtualization tool to run any software or program.

2. we can run a software instantly in a few min and close it when it is not required.

3. software runs inside a container in docker

4. container is like a virtual machine

5. Docker commands:

	To list all running containers
	docker ps 
	
	To list all running/stopped containers
	docker ps -a
	
	To stop a container
	docker stop <container_name>
	
	To start a container
	docker start <container_name>
	
	To run a container from an image (like mysql)
	docker run --name <container_name> -e <env_variable>=<value> -p <container_port>:<machine_port> -d mysql
	
	-e environment variables
	-p port mappings
	-d run container in detached mode
	
	
	To connect to a running container
	docker exec -it <container_name> <program_name> -u <user_name> -p
	-it run container in interactive mode

    To exit out of a container
    exit
